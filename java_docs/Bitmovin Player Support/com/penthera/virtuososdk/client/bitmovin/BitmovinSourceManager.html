Class BitmovinSourceManager

  • java.lang.Object
    • com.penthera.virtuososdk.client.bitmovin.BitmovinSourceManager
  • public class BitmovinSourceManager extends java.lang.Object
    This utility class helps to create and work with the bitmovin player when streaming content from the Penthera D2G SDK. To use this helper: 1) first create an instance of the class and keep it in the activity with the player. 2) Call getBitmovinSourceItem() to fetch the bitmovin source object that can be passed to the player. Do not do this on the main thread, as it may block while checking the DRM license is still valid. If the source cannot be generated, for instance if the license is not valid and cannot be renewed then the method returns null. 3) When using DRM protected assets, call startMonitoring() when the player starts and in onResume(). 4) When using DRM protected assets, call stopMonitoring() in onPause(). 5) When using DRM protected assets, call cleanup() when playback has finished, such as in onDestroy().

      public void startMonitoring()
      startMonitoring should be called when playback commences of a DRM protected asset, to ensure that additional files created in the player cache are cleaned up in a timely fashion to prevent consuming large volumes of disk storage during playback. Call this in your onResume() or a similar place in the lifecycle. stopMonitoring() should be called in a pair with this method when the playback has ceased.

Interface IPrepareURLObserver

  • public interface IPrepareURLObserver
    This interface provides a configurable mechanism to allow modification of the request URLs for segments or keys (HLS) prior to requests being made. The class implementing IPrepareURLObserver is instantiated within the service process of the SDK and is called directly within the downloading process. It may be called when the app is in the background as downloading continues while the app is backgrounded. It will only be called when the SDK determines that the device is online. The IPrepareURLObserver is provided to the engine through the IBackgroundProcessingManager interface, which itself is registered via the Android manifest.

      java.lang.String prepareEncryptionKeyURL(java.lang.String keyUrl, ISegment segment, ISegmentedAsset asset)
      This method is called directly prior to beginning download of each key within a segmented asset. It must return either an updated URL to use for the download, or null to continue with the existing url. This class should only be necessary if key URLs need to be updated dynamically at the point of download. If the urls can be updated at the point when the asset is added and the manifest parsed then it is more efficient to use the IManifestParserObserver. If this class is returned from the background processing manager then this method will be called for every segment download attempt, so it should be as efficient as possible in returning updated URLs. Note: the segment details provided to this method makes it possible to check if the segment download is being attempted for the first time or has failed a download previously.
      Parameters:
      keyUrl - The original key URL
      segment - The segment entry corresponding to the URL to be downloaded, contains information about previous download attempts
      asset - The asset to which the segment belongs.
      Returns:
      An updated download URL, or null to continue using the original URL provided.

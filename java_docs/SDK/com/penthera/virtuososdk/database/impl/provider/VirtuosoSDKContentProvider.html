    public abstract class VirtuosoSDKContentProvider extends android.content.ContentProvider implements android.os.Handler.Callback
    This is the base class which all client content providers should derive from. This abstract base class provides the support for enabling multiple clients usage of the Virtuoso Download SDK. To use this base class the client must provide an implementation that calls setAuthority in a static block and override the abstract getAuthority method to return the authority name as defined in the AndroidManifest file for the content provider and for the com.penthera.virtuososdk.client.pckg meta-data specified at the application level. An example of a Content Provider implementation is as follows: 
     
 package com.penthera.reference;

 import com.penthera.virtuososdk.database.impl.provider.VirtuosoSDKContentProvider;

 public class ReferenceContentProvider extends VirtuosoSDKContentProvider {

 static{
         setAuthority("com.penthera.reference.provider");
 }

 protected String getAuthority() {
         return "com.penthera.reference.provider";
    }

 }
    The Android manifest file will then contain the following for the above provider. 
     
 <meta-data android:name="com.penthera.virtuososdk.client.pckg" android:value="com.penthera.reference.provider" />
 <provider android:name="com.penthera.reference.ReferenceContentProvider"
                         android:authorities="com.penthera.reference.provider"
                         android:exported="true"
                         android:process=":remote"/>
    The content provider must be exported to insure Virtuoso functions correctly on a device with multiple installations of the Virtuoso service
